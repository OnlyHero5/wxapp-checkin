# 2026-02-03 Moonshot Miniapp UI Design (TDesign)

## 目标与范围
- 目标：以 Moonshot 极客极简美学重做小程序四个页面（签到、记录、详情、注册），并新增极简“个人中心”页与 TabBar 入口。
- 约束：保持现有功能流程与状态反馈；组件库切换为 TDesign 小程序（腾讯官方）。

## 视觉方向（Obsidian Grid）
- **基底**：黑底（#0b0d12 / #0f172a）+ 极轻微石墨纹理/细网格质感。
- **边界**：细边框（1rpx，半透明深灰），避免大阴影。
- **强调色**：冷绿（#65f0c2 / #22c55e）用于主 CTA、选中态、成功提示。
- **排版**：中文主字体使用 HarmonyOS Sans / 思源黑体系；数字与标签使用等宽风格。
- **节奏**：大留白 + 精准间距，减少装饰性元素。

## 组件库（TDesign 小程序）
- 包名：`tdesign-miniprogram`
- 推荐 NPM 安装，使用 `usingComponents` 方式引入组件。
- 组件引入示例：`"t-button": "tdesign-miniprogram/button/button"`
- 最低基础库版本：`^2.6.5`

## 设计令牌（映射到 app.wxss）
- 背景：`--bg` = #0b0d12
- 表面：`--surface` = #0f172a
- 边框：`--border` = rgba(148,163,184,0.18)
- 主文字：`--text` = #f8fafc
- 次级文字：`--muted` = #94a3b8
- 强调：`--accent` = #65f0c2
- 危险：`--danger` = #f87171
- 间距：8 / 16 / 24 / 32 / 48 rpx

## 组件映射（TDesign）
- Button：`t-button`
- Input：`t-input`
- Tag：`t-tag`
- Cell / List：`t-cell` + `t-cell-group`
- Avatar：`t-avatar`
- 其他视觉块使用原生 `view` + token 化样式（保持极简结构）

## 页面规格

### 1) 签到页
- 结构：标题区 + 活动信息区 + 主 CTA + 提示文案 + 状态卡 + 成功浮层。
- 交互：扫码后按钮进入加载态；成功弹层 + “查看记录”。
- 状态：网络异常顶部条、二维码失效/重复/身份不匹配提示卡。

### 2) 签到记录页
- 结构：标题区 + 列表（活动名、时间、地点、状态 Tag）。
- 交互：点击进入详情；空状态居中弱提示。

### 3) 签到详情页
- 结构：详情卡纵向信息；底部弱提示“可截图保存”。
- 交互：返回按钮为次级样式。

### 4) 注册绑定页
- 结构：身份状态 Tag + 学号/姓名输入 + 提交 CTA。
- 交互：提交时按钮加载；失败提示靠近按钮位置。

### 5) 个人中心（极简）
- 结构：头像占位 + 姓名/学号 + 身份状态 Tag + 单一入口按钮（绑定/重新绑定）。
- 不堆功能，仅保持“身份感知 + 操作入口”。

## 数据流与反馈
- 主流程：扫码 → 验证 → 成功/失败状态，按钮文案与样式同步变化。
- 失败提示：同一位置出现，文案短且清晰；网络异常禁用按钮。
- 记录页：列表与详情字段完整展示，保持截屏引导。

## 动效与交互
- 动效控制 150–250ms：按钮按压、成功弹层轻微缩放、页面淡入。
- 禁止高频闪烁与大范围位移，保持克制与专业感。

## 可用性与可访问性
- 文字对比度 ≥ 4.5:1。
- 所有可点击区域满足 44x44 触控尺寸。
- 状态变化不只依赖颜色（配合文字）。
