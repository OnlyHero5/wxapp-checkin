思路说明：
- 前端通过 `auth.ensureSession` 统一获取 `session_token`，避免各页重复逻辑。
- API 层提供 mock 模式，确保无后端时也能直接运行和演示流程；上线时将 `src/utils/config.js` 的 `mock` 改为 `false` 并配置 `baseUrl`。
- 加密逻辑暂用 Base64 作为占位，后续可替换为后端指定的 AES/RSA 实现。
